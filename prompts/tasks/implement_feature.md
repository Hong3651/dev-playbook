# 작업 프롬프트: 기능 구현

기능 구현을 AI에게 요청할 때 붙이는 작업 프롬프트다.
관련 파일만 기준으로 최소 수정하게 하고, 부족한 파일은 먼저 요청하게 한다.

## 조립

```text
prompts/base.md
prompts/tasks/implement_feature.md
필요한 prompts/addons/*.md
구현 요청과 관련 파일
```

## 프롬프트

```text
이번 작업의 목표는 기능 구현이다.
하지만 컨텍스트가 부족하면 바로 구현하지 말고 필요한 파일을 먼저 요청해라.

입력:

- 구현할 기능:
- 기대 동작:
- 제외할 동작:
- 프로젝트 성격 또는 도메인:
- 적용할 addon:
- 제공한 파일 목록:
- 수정하면 안 되는 파일:
- 실행 또는 테스트 명령:

작업 방식:

1. 제공된 파일로 구현 가능한지 먼저 판단한다.
2. 부족하면 구현하지 말고 필요한 파일을 요청한다.
3. addon 기준이 부족하면 구현하지 말고 필요한 기준을 질문한다.
4. 변경이 크거나 위험하면 수정 전 설계안을 먼저 제시한다.
5. 관련 파일만 최소 수정한다.
6. 수정 이유를 파일별로 설명한다.
7. 테스트 방법과 수동 검증 방법을 제시한다.
8. 남은 위험을 정리한다.

출력 형식:

- 컨텍스트 충분 여부:
- 추가로 필요한 파일:
- 필요한 추가 기준:
- 수정 계획:
- 변경 내용:
- 테스트 방법:
- 남은 위험:
```
