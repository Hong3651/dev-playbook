# 작업 프롬프트: 설계 검토

설계안이 구현 전에 적절한지 검토할 때 붙이는 작업 프롬프트다.
코드를 작성하지 않고 위험과 빠진 판단을 찾게 한다.

## 조립

```text
prompts/base.md
prompts/tasks/design_review.md
필요한 prompts/addons/*.md
검토할 설계안
```

## 프롬프트

```text
이번 작업의 목표는 설계 검토다.
아래 설계안을 기준으로 검토만 해라.
코드를 작성하지 마라.
구현 방법을 길게 제안하기보다 위험 요소와 빠진 판단을 먼저 찾아라.

입력:

- 설계안:
- 프로젝트 성격 또는 도메인:
- 적용할 addon:
- MVP 범위:
- 제외 범위:
- 특별히 걱정되는 부분:

검토 기준:

1. 과설계 여부
2. 빠진 위험 요소
3. 프로젝트 성격과 설계가 맞는지
4. MVP 범위가 적절한지
5. 공통 개발 원칙 위반 가능성
6. addon 기준 누락 여부
7. 테스트 가능성
8. 민감정보 노출 가능성
9. 추가로 필요한 기준

출력 형식:

- 전체 판정: 통과 / 조건부 통과 / 재검토 필요
- 프로젝트 성격 적합성:
- 주요 문제:
- 과한 구조:
- 빠진 위험 대응:
- MVP 범위 조정:
- 필요한 추가 기준:
- 확인 질문:
```
