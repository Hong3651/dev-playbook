# 작업 프롬프트: 새 프로젝트 시작

새 프로젝트를 시작할 때 붙이는 작업 프롬프트다.
바로 코드를 만들게 하지 않고 목적, 범위, 위험, 필요한 문서를 먼저 정리하게 한다.

## 조립

```text
prompts/base.md
prompts/tasks/start_project.md
필요한 prompts/addons/*.md
실제 프로젝트 설명
```

## 프롬프트

```text
이번 작업의 목표는 새 프로젝트 시작을 정리하는 것이다.
아직 코드를 작성하지 마라.

입력:

- 프로젝트 목적:
- 사용자 또는 사용 상황:
- 반드시 포함할 기능:
- 이번에 제외할 기능:
- 도메인 또는 프로젝트 타입:
- 적용할 addon:
- 예상 실행 환경:
- 민감정보 포함 여부:

작업:

1. 프로젝트 성격을 사용자의 설명 기준으로 정리한다.
2. 성격이 애매하면 복수 후보와 이유를 말한다.
3. MVP 범위를 정리한다.
4. 제외 범위를 정리한다.
5. 초기 폴더 구조를 최소한으로 제안한다.
6. 먼저 작성할 문서를 제안한다.
7. 위험 요소와 아직 모르는 질문을 정리한다.
8. 필요한 addon이나 도메인 기준이 부족하면 질문한다.

출력 형식:

- 프로젝트 성격:
- 판단 이유:
- MVP 범위:
- 제외 범위:
- 초기 구조:
- 먼저 작성할 문서:
- 위험 요소:
- 필요한 추가 기준:
- 확인 질문:
```
