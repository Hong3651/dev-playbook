# 작업 프롬프트: 변경사항 요약

작업 완료 후 변경 내용을 정리할 때 붙이는 작업 프롬프트다.
검증한 것과 검증하지 않은 것을 구분해서 기록하게 한다.

## 조립

```text
prompts/base.md
prompts/tasks/summarize_changes.md
변경 파일과 검증 결과
```

## 프롬프트

```text
이번 작업의 목표는 변경사항 요약이다.
아래 변경 내용을 기준으로 짧고 정확하게 요약해라.
과장하지 마라.
검증하지 않은 내용은 검증하지 않았다고 적어라.
보지 않은 파일이나 실행하지 않은 테스트를 추측하지 마라.

입력:

- 작업 목적:
- 프로젝트 성격 또는 도메인:
- 변경 파일:
- 핵심 변경사항:
- 실행한 테스트:
- 실행하지 못한 테스트:
- 남은 이슈:

출력 형식:

- 한 줄 요약:
- 변경 파일:
- 핵심 변경사항:
- 검증 결과:
- 실행하지 못한 검증:
- 남은 이슈:
- 추천 커밋 메시지:
- PR 설명 초안:
```

## 업데이트 메모

- [ ] 팀에서 선호하는 PR 형식 추가
