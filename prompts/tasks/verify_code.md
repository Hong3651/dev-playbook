# 작업 프롬프트: 코드 검증

구현 결과를 별도 검증 에이전트처럼 확인할 때 붙이는 작업 프롬프트다.
새 기능 제안보다 결함, 누락, 위험, 테스트 부족을 찾게 한다.

## 조립

```text
prompts/base.md
prompts/tasks/verify_code.md
필요한 prompts/addons/*.md
요구사항, 변경 요약, 제공 파일
```

## 프롬프트

```text
이번 작업의 목표는 코드 검증이다.
아래 요구사항, 변경 내용, 제공된 파일만 기준으로 검증해라.
새 기능을 제안하기보다 결함, 누락, 위험, 테스트 부족을 우선해서 찾아라.

입력:

- 요구사항:
- 프로젝트 성격 또는 도메인:
- 적용할 addon:
- 변경 파일:
- 변경 요약:
- 테스트 결과:
- 제공된 파일:

검증 기준:

1. 요구사항 만족 여부
2. 런타임 에러 가능성
3. 예외 처리
4. 테스트 가능성
5. 설정값 하드코딩
6. 민감정보 노출 가능성
7. 외부 의존성 실패 대응
8. 기존 동작 회귀 가능성
9. addon 기준 만족 여부
10. 검증에 필요한 정보가 충분한지

출력 형식:

- 전체 판정: 통과 / 조건부 통과 / 실패
- 치명적 문제:
- 일반 문제:
- 테스트 부족:
- addon 기준 검증:
- 검증 불가 항목:
- 확인 질문:
- 권장 수정 순서:
```
