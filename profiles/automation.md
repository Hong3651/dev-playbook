# 프로필: 자동화 스크립트

파일 처리, 로그 정리, 데이터 변환, 실험 보조, 반복 작업 자동화처럼 작게 만들고 빠르게 검증하는 작업에 적용한다.

## 선택 기준

이 프로필을 고르는 경우:

- 사람이 반복하던 작업을 스크립트로 줄인다.
- 입력과 출력이 파일, 폴더, 로그, 표, 데이터셋처럼 비교적 명확하다.
- 실패해도 물리 장비나 운영 서비스에 직접 영향을 주지 않는다.
- 빠르게 만들고 샘플 입력으로 검증하면 충분하다.

이 프로필만으로 부족한 경우:

- 실제 장비, 센서, 액추에이터, inference loop와 연결된다면 `robot_infra.md`를 함께 본다.
- 사용자가 웹 화면이나 API를 통해 접근한다면 `web_service.md`를 함께 본다.
- 여러 사람이 장기간 운영하고 권한, 저장소, 배포가 필요하면 별도 프로젝트 구조를 잡는다.

애매한 경우에는 실패 시 영향으로 판단한다.
파일을 다시 만들면 되는 수준이면 automation, 장비 동작이나 사용자 요청 처리에 영향을 주면 다른 profile을 같이 적용한다.

## 적용 범위

- 일회성 또는 반복 자동화 스크립트
- 파일명 변경, 포맷 변환, 로그 파싱
- 데이터셋 보조 처리
- 실험 실행 전후 정리 작업
- 사람이 수동으로 반복하던 작업 자동화

## 기본 기준

- 단일 파일을 허용한다.
- 과한 객체지향을 피한다.
- 입력, 출력, 실패 조건을 먼저 정한다.
- 경로와 설정값은 상수, 실행 인자, 설정 파일 중 하나로 분리한다.
- 실행 방법과 예시는 README 또는 파일 상단에 기록한다.

## 단일 파일로 충분한 경우

- 입력과 출력이 단순하다.
- 외부 API, DB, 장비 연결이 없다.
- 혼자 사용하거나 사용자가 적다.
- 실패해도 복구가 쉽다.

## 구조를 키워야 하는 경우

- 기능이 여러 개로 나뉜다.
- 테스트가 필요하다.
- 설정값이 많아진다.
- 외부 API, DB, 장비 연결이 생긴다.
- 여러 사람이 반복 사용한다.

이 경우 별도 프로젝트 구조를 만들거나 `robot_infra.md`, `web_service.md` 기준을 함께 본다.
